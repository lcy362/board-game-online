# 极速 101 · 视觉优化方案（画面清晰度 + 局势一目了然）

> 文档目标：针对现有单文件原型 `speed101/index.html`（v3，74 张牌、9 种功能牌、单人 vs AI）的 5 大视觉痛点，给出"哪个元素怎么画 / 怎么动"的落地方案。
> 设计原则：**清晰优先 > 装饰**；所有视觉语言必须**贴合游戏规则**；区分度**不只靠颜色**（色盲可辨）；给出 CSS/SVG 可实现路径（不写完整代码，只给方向）。
> 本文档与现有代码的对应关系见文末「§8 现有代码迁移映射」。

---

## 0. 现状诊断（基于实际代码）

| 痛点 | 现状代码表现 | 根因 / 问题 |
|---|---|---|
| ① 速度表看不懂 | `.speedometer::before` 用 `conic-gradient(from 180deg, …0deg→200deg)`（200° 色环），但指针用 `-120deg + pct*240`（-120°→+120°，240° 扫角）。两者坐标系统不一致 → **色环与指针数值不对应**，色环只是装饰。无 101 线、无刻度数字、`danger` 仅在 `>=90` 才红色脉冲。 | 色环未映射到真实数值刻度，玩家无法"一眼读出还差多少爆胎"。 |
| ② 排布无竞速感 | `.racers` 是 `display:flex; justify-content:space-between` 的**横排 emoji 小车**；`.racer.active` 仅上浮 4px + 黄边。出牌方向只在浮动提示里写"顺时针 ↻"。 | 无赛道、无方向流、无座位顺序的空间语义。 |
| ③ 功能牌靠颜色 | 已分 8 色（func-brake 青 / func-fuel 橙红 / func-skip 紫 / func-reverse 粉 / func-double 红 / func-extra 金 / func-flip 靛 / func 默认黄），配 emoji 图标 + 中文名。 | 区分**主要依赖颜色**；缺统一"形状/语义"层；色弱玩家难记。且多张卡文字对比度不达标（见 §6）。 |
| ④ 事件反馈弱 | `burstFx()` 仅给速度表加 `shake` + `beep()`；日志文字 + 轻微震动。跳过 / 升档 / 掉头 / 连续油门**无专属强反馈**。 | 关键状态变化"看不见"，新手不知道发生了什么。 |
| ⑤ 方向无指示 | 仅 `showFloatingTip` 文字提示；无持久可视的方向流。 | 掉头反转后，玩家难以感知方向已变。 |

**→ 核心结论**：清晰度问题集中在"仪表盘数值不可读"和"局势空间关系缺失"两处，应作为 P0 优先解决。

## 1. 核心设计原则

1. **数值可视化优先**：任何"速度多少 / 离爆胎多远"必须靠**位置 + 数字 + 颜色带**三重表达，不靠记忆。
2. **机制驱动**：每个视觉元素对应一条规则（速度带=爆胎阈值；赛道流=出牌方向；印章=被跳过）。不为好看而加无关装饰。
3. **色盲可辨（CB-safe）**：颜色只是**第二层强化**，第一层必须是**形状 / 图标 / 文字标签**。红绿不作为唯一编码对（deuteranopia 约占男性 6%）。
4. **状态即反馈**：重要事件用"强对比 + 动效 + 文字"同时发生，确保即使关掉动效也能读懂（见 §6 reduced-motion）。

## 2. 速度表重设计（SVG 仪表盘）

**策略**：弃用当前不准确的 `conic-gradient` 色环，改为**内联 SVG 仪表盘**（viewBox `0 0 200 200`，中心 `(100,100)`），让色带、刻度、指针全部基于同一套角度映射，从而与现有指针逻辑完全对齐。

### 2.1 角度映射（复用现有逻辑）
- 指针角度：`ang(s) = -120 + (s/101)*240`（度），`0°`=正上方，顺时针为正。
- 与现有 `renderSpeedometer()` 的 `angle = -120 + pct*240` 完全一致 → 迁移零成本。

### 2.2 三色带（安全 / 警戒 / 危险）+ 爆胎线
| 区段 | 速度范围 | 色（CB-safe） | 视觉 |
|---|---|---|---|
| 安全区 | 0 – 69 | 青蓝 `#2cb5ac` | 实色弧 |
| 警戒区 | 70 – 89 | 琥珀 `#f0a500` | 实色弧 |
| 危险区 | 90 – 101 | 红 `#e23b3b` | 实色弧 + 临近脉冲 |
| **爆胎线** | **101** | **红 + 黑描边** | **加粗径向红线 + "101 💥"标签，speed≥95 时闪烁** |

- **实现路径 A（推荐）**：用 3 段 SVG `<path>` 弧（stroke 宽度 ~16，半径 R=82）拼出色带，每段起止点用 `ang()` 计算。色带即"还剩多少余量"的直观读图。
- **实现路径 B（兜底）**：若想保留 conic-gradient，须改为覆盖 240° 扫角并起点对齐（计算繁琐、易错，**不推荐**）。
- **爆胎线**：在 `ang(101)=+120°` 处画一条从 R-4 到 R+14 的粗红线，末端放 `<text>101</text>`；加 `.blow-line` 类，`@keyframes blink {50%{opacity:.3}}`，`speed>=95` 时挂上。

### 2.3 刻度与数字
- 主刻度：`[0, 25, 50, 75, 101]` 五档，每档画短刻度线（R 到 R+10）+ 文字标签（R+22）。
- 次刻度（可选）：每 10 一档更细的线，增强"读秒"感。
- **中心双重反馈**：保留大号数字 `speed-value`（现 3.2rem 900 字重），其下方新增**状态词**"安全 / 警戒 / 危险"，颜色随区段变化（绿/琥珀/红）——数字 + 位置 + 文字，三重确认。

### 2.4 指针
- SVG `<g id="needleG"><polygon …/></g>`，多边形从中心画到 `(100,18)`（朝上），用 `transform="rotate(ang,100,100)"` 旋转。
- JS 迁移：`needleG.setAttribute('transform','rotate('+ang+',100,100)')` 替换原 `needle.style.transform`。
- 过渡：`transition: transform .4s cubic-bezier(.34,1.56,.64,1)`（沿用现有缓动，带轻微回弹，更有"表针"感）。

### 2.5 爆胎时的强反馈
- 危险区色带在 `speed>=90` 起 `pulse`；**爆胎事件**触发时：危险红带做**一次强烈脉冲**（比普通 pulse 幅度更大、更短促），指针"啪"地回弹到 0（加一个 `recoil` 关键帧：先过冲到 +130° 再回 0）。
- 同时联动 §5 的全局红闪与震屏。

## 3. 赛道式布局（椭圆轨道 + 方向流）

**策略**：把横排小车换成**椭圆 / 体育场形赛道**，玩家与 AI 按座位顺序分布在赛道上，方向用"流动光带"表达。

### 3.1 轨道（SVG 底层）
- 容器 `position:relative; aspect-ratio: 16/10`（移动端竖屏也不至于太扁）。
- 底层 SVG 画两条轨道线：外圈粗灰"沥青"（`#3a3f4b`，stroke ~14）+ 内侧虚线分道线。
- 尺寸：viewBox 适配容器，`cx,cy` 为中心，`Rx, Ry` 为椭圆半轴（如 Rx=42%, Ry=34% 相对 viewBox）。

### 3.2 车手节点定位（DOM 叠加在 SVG 上）
- 在相对定位容器内，按座位顺序绝对定位每个车手节点（emoji 车 + 名字 + 轮胎）：
  ```
  t  = (i / total) * 2π - π/2          // 从正上方开始
  x  = cx + Rx * cos(t)
  y  = cy + Ry * sin(t)
  node.style.left = x + 'px'; node.style.top = y + 'px'
  ```
- `total` = 存活+淘汰的玩家总数（2–6），位置固定不随淘汰漂移（淘汰者变灰，不重排，避免"谁在哪"混乱）。
- **增强（可选）**：把车 emoji 按 `rotate(t + π/2)` 旋转，使其"车头朝前进方向"，强化竞速感。

### 3.3 方向流光带（核心：让"顺 / 逆"可见）
- 在赛道线上叠加一条**流动虚线** `<path class="flow">`，`stroke-dasharray: 12 14; animation: flow 1s linear infinite`。
- `@keyframes flow { to { stroke-dashoffset: -26 } }`（顺时针）；逆时针时给容器加 `.reverse` 类，用 `animation-direction: reverse` 反向流动。
- 沿轨道再放 2–3 个**箭头楔形**（chevron）随流动方向"漂"，强化方向语义。
- 顶部居中放一个**方向徽章**：`↻ 顺时针` / `↺ 逆时针`，掉头时徽章做 360° 翻转过渡。

### 3.4 当前玩家 / 下家高亮
- **当前玩家（你 / 当前 AI）**：节点加 `.current`——亮色描边光环 + 车头前方一个 `▶` 或 🚩 指向标 + 轻微上浮，明确"现在轮到谁"。
- **下家（nextAliveIdx）**：节点加 `.next`——**脉冲呼吸环**（柔和，不抢戏），预览"下一个是谁"，呼应出牌方向。

### 3.5 迁移
- 改 `renderRacers()`：不再生成 flex 子项，改为在轨道容器内按上述公式绝对定位节点；每帧根据 `state.currentIdx` / `nextAliveIdx()` 切换 `.current` / `.next`；根据 `state.direction` 切换 `.reverse`。

## 4. 功能牌视觉系统（统一图标语义 + 形状编码）

**问题**：现状 8 色 + emoji 已不错，但**区分度以颜色为主**，色弱玩家难记；且部分卡文字对比度不达标。
**方案**：建立**三层语义**——① 图标（主，色盲可辨）② 形状/角标（次，不需颜色）③ 颜色（强化）。每层单独都能区分。

### 4.1 统一图标语义规范
- 所有功能牌**必须**含：大号图标 + 中文名 + 效果数值（如"刹车 -10"）。图标优先用**单色描边 SVG**（可灰度仍可读），emoji 作为 v1 快速落地备选。
- **形状 / 角标编码（不依赖颜色即可分辨）**：
  - 减速类 → **向下双箭头 ▼▼** + 左上角**红色三角警示**
  - 加速 / 能量类 → **向上箭头 ▲** 或 油枪形
  - 控场 / 规则类 → **环形箭头 ↻ / 闪电 ⚡ / 靶心 🎯**

### 4.2 九种功能牌规格表（CB-safe）

| 牌名 | 效果 | 主图标(语义) | 形状/角标 | 颜色(强化) | 色盲安全配对 |
|---|---|---|---|---|---|
| 刹车 -10 | add -10 | 🛑 停止标志 | 向下双箭头 + 红三角 | 红 `#e23b3b` | 图标+向下箭头，绝不单靠红 |
| 急刹 -20 | add -20 | 🛑⚠️ 停止+警示 | **双红三角 + 黑粗边** | 深红 `#b91c1c` + 黑描边 | 与"刹车"靠⚠️+黑边区分，非仅深浅 |
| 加油 +18 | add +18 | ⛽ 油枪 | 向上箭头 | 橙 `#e8830c` | 油枪形唯一，无需颜色 |
| 调速 50 | set 50 | 🎯 靶心 | 同心圆纹 | 蓝 `#1f6feb` | 靶心形唯一 |
| 超车 | skip 下家 | 💨 气流/超车 | 前向箭头 | 紫 `#7c3aed` | 气流+箭头语义 |
| 掉头 | reverse 反转 | ↩️ U 形弯 | 回转弧线 | 粉 `#db2777` | U 弯形唯一 |
| 连续油门 | double 连出2 | ⚡⚡ 双闪电 | 连锁折线 | 橙红 `#ea580c` | 双闪电形唯一 |
| 升档加速 | extraSpeed | ⬆️ 升档箭头/齿轮 | 向上粗箭头 | 金 `#ca8a04` | 升档箭头唯一 |
| 失控反转 | flip 101-速 | 🔄 翻转/刷新 | 双向弧箭头 | 靛 `#4f46e5` | 翻转形唯一 |

> 关键点：**红家族（刹车 / 急刹 / 连续油门）不靠"红=停"单一语义**，而用 🛑 / 🛑⚠️ / ⚡⚡ 的形状差异 + 黑边/双三角区分；中文名始终显示。

### 4.3 卡面结构（统一模板）
- 顶部一条**彩色 header 条**（承载颜色编码），主体为**浅底深字**（解决 §6 对比度）；左下角放**形状角标**，中央大图标 + 名称 + 数值。
- 速度牌维持白底大数字，但负数用青蓝、正数用红（沿用现有 `.pos/.neg` 思路，并加 ▲/▼ 小标强化方向）。
- 统一 `min-width:72px`（见 §6），图标区占比 ≥ 40%，保证小屏也认得出。

## 5. 事件动效（强视觉反馈）

统一的 FX 层：一个 `position:fixed; inset:0; pointer-events:none; z-index:60` 的 `#fx` 覆盖层，用于全屏闪 / 粒子；节点级动效用挂 class 实现。

| 事件 | 视觉 | 实现路径（CSS / SVG 方向） |
|---|---|---|
| **爆胎** | ① 轮胎碎裂（🛞→💥 + 3–5 片 SVG 碎片向外飞散淡出）② **全屏红闪**（`#fx` 红层 opacity 0→.55→0，.18s）③ **震屏**（`#app` 加 `.shake`，幅度比现 `burstFx` 更强）④ 联动 §2.5 红带强脉冲 + 指针回弹 | 碎片用绝对定位小 `<div>`+`@keyframes shatter{to{transform:translate(x,y) rotate();opacity:0}}`；红闪/震屏见 §6 reduced-motion 例外处理 |
| **被跳过** | 灰色**"跳过 / SKIP"印章**盖在被跳过车手节点上（旋转 -12° 砸下：`scale(1.4)→1` + 透明度），该节点变灰 | 节点上挂 `.stamp`，伪元素画圆环+文字；`animation: stamp .35s` |
| **升档加速** | 玩家区域**金色流光**扫过 + "升档"徽章发光；随后打出的速度牌带金色拖尾 | `linear-gradient` 金层 `translateX(-100%)→100%` 循环一次；牌加 `.gold-trail` |
| **连续油门** | 当前玩家 → 下家之间画**红色连锁闪电**（SVG polyline 闪 2 次），下家节点红色脉冲 + "连出 2 张"横幅 | 闪电用 `<polyline>` + `@keyframes bolt{50%{opacity:1}}` 闪；横幅复用 `showFloatingTip` 样式改红 |
| **掉头（方向反转）** | 整条赛道**翻转过渡**（流向虚线反向 + 方向徽章 ↻/↺ 360° 旋转），玩家节点轻微"甩尾" | 轨道容器加 `.reverse`（流光 `animation-direction:reverse`）；徽章 `rotate(360deg)` 过渡 |

- 每个事件**同时**写一条带类型的日志（现有 `log(...,'event')` 已支持），保证动效关闭时仍可读。
- 音效：爆胎用低频长 `beep(80,400,'sawtooth')`；升档用上行音；跳过用短促闷响——区分事件（可选 P2）。

## 6. 可访问性规范

### 6.1 文字对比度（修复现状不达标项）
现状多张功能牌为白字（`#fff`）压在中等明度渐变上（如 func-brake `#fff`/`#2cb5ac` 约 2:1，**远低于 WCAG AA 4.5:1**）。修复三选一：
- **推荐**：卡面主体改**浅底深字**（`#1f2937` 深字压浅色卡身，对比 ≥ 7:1），颜色只出现在顶部 header 条 → 同时解决美观与对比。
- 或：彩色卡渐变末端压暗到 `#1d8a82` 级别 + 文字加 `text-shadow:0 1px 2px rgba(0,0,0,.4)`（仅对大字粗体勉强达 3:1）。
- 正文最小 `14px`（`.85rem`），关键数字 ≥ `1.6rem` 且 900 字重。

### 6.2 最小点击区 72px
- 现状 `@media(max-width:360px)` 把 `.hand-card` 缩到 `64×86`，**宽度低于 72px 底线**。
- 修复：`.hand-card{ min-width:72px }`；小屏**不缩小卡**，改为**横向滚动**手牌区（`overflow-x:auto`）或减小卡间距（gap 8→6），保证每张可点区 ≥72px 且卡间留 ≥8px 防误触。

### 6.3 色盲友好配色
- 主编码对用**蓝 `#1f6feb` ↔ 橙 `#e8830c`**（deuteranopia/protanopia 下仍可分）；红 `#e23b3b` 仅用于"危险/爆胎"且**必配图标+文字**。
- 速度表三色带用 §2.2 的青蓝/琥珀/红，并均有刻度与文字辅助。
- 提供**色盲模式开关**（P2）：开启后功能牌去掉彩色、仅留图标+形状+文字（验证 CB-safe）。

### 6.4 动效偏好 & 辅助技术
- **Reduced motion**：`@media (prefers-reduced-motion: reduce){ .shake,.pulse,.flow,.stamp,.bolt,.gold-trail{animation:none!important} }`——**保留状态变化**（颜色、印章、标签、数字），只去运动。
- **屏幕阅读器**：`#log` 加 `aria-live="polite"`，回合/事件自动播报；每张卡 `role="button" tabindex="0"` + `aria-label="功能牌 刹车 减10"` / `"速度牌 加5"`。
- **键盘可玩**：卡片支持 `Enter/Space` 出牌，`←/→` 移动焦点（P2，桌面/无障碍必需）。

## 7. 落地优先级（P0 / P1 / P2）

| 优先级 | 项目 | 对应痛点 | 工作量 | 风险 | 理由 |
|---|---|---|---|---|---|
| **P0** | ① 速度表 SVG 化：101 爆胎线 + 三色带 + 刻度数字 | ① | 中 | 低 | 直接修复"看不懂速度"的核心问题；复用现有角度逻辑 |
| **P0** | ② 方向流光带 + 方向徽章（赛道可先不做，流光可叠在现有横排上） | ⑤ | 小 | 低 | 改动小、收益高，让"顺/逆"持久可见 |
| **P0** | ③ 功能牌对比度修复 + 最小点击 72px | ③/⑥ | 小 | 低 | 纯 CSS 修正，合规且提升可读性 |
| **P1** | ④ 椭圆赛道布局 + 当前/下家高亮 | ② | 中 | 中 | 视觉大改，需在小屏验证定位；建议 A/B 或开关 |
| **P1** | ⑤ 功能牌"图标+形状"统一系统（9 卡规格表） | ③ | 中 | 低 | 提升记忆与色盲可辨 |
| **P1** | ⑥ 事件动效：爆胎强 FX / 跳过印章 / 升档流光 / 掉头翻转 | ④ | 中 | 低 | 强反馈，显著改善"发生了什么" |
| **P2** | ⑦ 连续油门连锁闪电 + 粒子细化 | ④ | 小 | 低 | 锦上添花 |
