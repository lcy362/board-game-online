# 极速101 · 体验优化统一方案（画面 + 状态展示）

> 版本 v1 · 2026-07-18 · 汇编自文策渊（UX）与林绘澄（美术）两份专家方案
> 目标：**让玩家更清晰理解规则 + 更清晰掌握当前现状**
> 范围：仅优化现有 `speed101/index.html` 原型的交互/视觉层，不改动核心规则数值（爆胎101、3轮胎、方向反转、连续油门等机制不变）

---

## 0. 现状诊断（两个方案共同确认的核心问题）

1. **速度表读不懂**：`.speedometer::before` 用 `conic-gradient(from 180deg …0deg→200deg)`（200° 色环），而指针用 `-120 + pct*240`（240° 扫角）——两套坐标不一致，色环只是装饰，玩家读不出"还差多少爆胎"。且无 101 爆胎线、无刻度、无余量。
2. **局势空间关系缺失**：横排 emoji 小车看不出顺/逆时针、看不出下家。
3. **手牌不知后果**：只着色，不知道"打出会到多少速度"、哪张安全。
4. **模式提示易错过**：连续油门/升档/掉头仅靠 2 秒浮动提示。
5. **事件无强反馈**：爆胎/跳过/升档/掉头/连续油门无专属视觉。
6. **可访问性不达标**：部分功能牌白字压中明度渐变（对比度≈2:1，远低于 WCAG AA 4.5:1）；小屏卡宽 64px < 72px 点击底线。

## 2. 手牌三态计算（引擎忠实，无虚构机制）

设 `S = state.speed`，预测 `S'`（不改变速度的 4 种功能牌无 `S'`）：

| 牌 | 公式 | 说明 |
|---|---|---|
| 速度牌 +v | `S' = S + v` | v∈[1,12] |
| 刹车 -10 | `S' = max(0, S-10)` | 不为负 |
| 急刹 -20 | `S' = max(0, S-20)` | 不为负 |
| 加油 +18 | `S' = S + 18` | |
| 调速50 | `S' = 50` | 恒安全 |
| 失控反转 | `S' = 101 - S` | S=0 时临界 101 |
| 超车/掉头/连续油门/升档 | 不改 S | 策略牌 |

**阈值**：危险区上界取 **85**（与 `chooseAiCard` 中 `danger = state.speed >= 85` 对齐）。`S'=101` 不爆（引擎 `>101` 才爆），标"危险(临界)"。

## 4. 已确认的 4 项决策（2026-07-18 用户拍板）

1. **必爆牌交互**：✅ **灰标 + 点击确认弹窗**（规避"全手牌必爆"软锁，保留合法操作）
2. **危险阈值**：✅ **85**（与 AI `danger>=85` 对齐；速度表危险脉冲阈值同步从 90 改为 85）
3. **AI 思考气泡**：✅ **默认开**（用户决定：所有难度均显示 AI 意图气泡，含简单模式；按原 P2-1 排期实现）
4. **独立无障碍文档**：✅ **不必**（方案已内嵌对比度/72px/CB-safe/reduced-motion/aria 原则）
