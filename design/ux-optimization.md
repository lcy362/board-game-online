# 极速101 · UX 优化方案

> **版本**：v1 · 2026-07-18
> **作者**：文策渊（design-strategist）
> **关联文档**：`/Users/lcy/speed101/speed101/index.html`（现有单文件原型）、`/Users/lcy/speed101/极速101改造方案.md`
> **核心目标**：让玩家**更清晰理解规则**与**更清晰掌握当前现状**，降低新手门槛、减少误操作与“看不懂在发生什么”的挫败感。
> **范围**：仅针对现有 H5 单文件原型的交互/信息层做优化设计，不改动核心规则引擎数值（爆胎 101、3 轮胎、方向反转、连续油门连出等机制保持不变）。

---

## 0. 目标与设计原则

### 0.1 两条设计支柱（不可漂移）
1. **现状一目了然**：任何时刻玩家都能答出“我现在速度多少、离爆胎还有多少、轮到谁、下家是谁、当前是什么模式”。
2. **规则无需背**：每张功能牌自带“图标 + 一行效果 + 何时用”，点一下就能看说明；首局有互动引导。

### 0.2 设计红线（自检用）
- **杜绝认知过载**：状态标注只用 3 个视觉档（安全/危险/必爆）+ 1 个策略档，不堆更多颜色。
- **不靠颜色 alone 区分**：色盲用户必须能靠图标/文字区分三态（✓ / ! / ✕）。
- **不制造软锁**：任何状态下玩家都必须有可执行的合法操作（见 §5 决策点）。
- **机制忠实**：所有展示严格对应引擎真实逻辑（`state.speed`、`state.direction`、`state.doublePlayTarget`、`state.extraSpeedMode`、`p.tires`），不展示引擎没有的状态。

## 3. 详细方案

### 3.1 状态展示清晰度

#### 3.1.1 手牌「预计到达速度」三态标注 【P0】
**区域**：`#hand`（`renderHand` 改造）
**目的**：让玩家出牌前就知道“打出这张我会到多少速度、安不安全”。

**计算逻辑（公式，单位：速度值，无量纲）**
设当前速度 `S = state.speed`，打出牌后的预测速度 `S'`：

| 牌类型 | 公式 | 备注 |
|---|---|---|
| 速度牌 `+v` | `S' = S + v` | v ∈ [1,12] |
| 刹车 `-10` | `S' = max(0, S - 10)` | 不为负 |
| 急刹 `-20` | `S' = max(0, S - 20)` | 不为负 |
| 加油 `+18` | `S' = S + 18` | |
| 调速50 | `S' = 50` | 与 S 无关，恒安全 |
| 失控反转 | `S' = 101 - S` | S=0 时 S'=101（临界） |
| 超车 / 掉头 / 连续油门 / 升档 | 不改变 S | **策略牌**，无 `S'` |

**三态判定（基于 `S'`）**
| 状态 | 条件 | 视觉（颜色 + 图标 + 文字） | 可点击 |
|---|---|---|---|
| ✅ 安全 | `S' < 85` | 绿底 + ✓ + “到 X” | 是 |
| ⚠️ 危险 | `85 ≤ S' ≤ 101` | 橙红闪边框 + ! + “到 X（临界）” | 是 |
| ✕ 必爆 | `S' > 101` | 灰底 + 红闪 + ✕ + “到 X（爆胎）” | 见 §5 决策点 |
| 🔵 策略 | 不改变 S 的 4 种功能牌 | 蓝底 + 标签（“不改速度 / 跳过下家 / 反转 / 连出2张 / 额外出牌”） | 是 |

> 阈值说明：引擎爆胎判定为 `state.speed > MAX_SPEED`（严格大于 101），故 `S' = 101` **不爆**，归入“危险（临界）”上界，与引擎一致。
> “危险”阈值取 **85**，与 `chooseAiCard` 中 `danger = state.speed >= 85` 对齐，避免玩家看到的危险区与 AI 行为逻辑脱节。

**组合数下文处理（`renderHand` 需注意）**
- 升档激活中（`state.extraSpeedMode && isTurn`）：仅速度牌可出；升档牌本身显示“额外出牌”而非速度数。
- 连续油门目标中（`state.doublePlayTarget === p.id && isTurn`）：必须出牌，所有手牌（含策略牌）均可出，顶部横幅提示剩余张数。
- 非自己回合：全部 `disabled`，但**仍显示**三态颜色（让玩家提前规划下一手）。

**视觉规范（不新增颜色，复用现有调色板）**
- 安全绿：沿用 `--safe #0a9c7c / #4ecdc4`
- 危险：沿用 `--danger #ff6b6b`，加 `animation: pulse`（复用现有 `@keyframes pulse`）
- 必爆灰：沿用 `--disabled` 灰度 + 红闪
- 策略蓝：沿用 `#74b9ff / #0984e3`
- 每张卡底部固定一行小字：“到 **X**”（X = `S'`），色随状态变。

#### 3.1.2 速度表增强：爆胎红线 + 余量 + 安全/危险区 【P0】
**区域**：`#speedometer` / `#speed-value` / `#needle`（`renderSpeedometer` 改造）

1. **爆胎红线（101）**：在指针最大角度 `+120°`（即 101 对应位置）画一条加粗红色刻度线 + 文字标签“爆胎线 101”。现有 `conic-gradient` 已有红区，但需补**明确的 101 边界标记**。
2. **距爆胎余量**：在表盘下方加一行读数：`距爆胎余量 = 101 - S`（单位：速度值）。例：S=73 → “距爆胎 28”。
3. **区带标签**：把现有无标签 `conic-gradient` 改为三段并加文字：
   - 安全区 `0–69`（青/绿）
   - 警示区 `70–84`（黄 `#ffd93d`）
   - 危险区 `85–101`（红 `#ff6b6b`）
   - 在表盘外环三个区域旁标注“安全/警示/危险”小字。
4. **危险脉冲对齐**：把现有 `if (state.speed >= 90)` 改为 `>= 85`，与手牌三态、AI 危险阈值统一。

#### 3.1.3 出牌顺序指示：下家箭头 + 实时更新 【P0】
**区域**：`#racers`（`renderRacers` 改造）

- **方向环**：在玩家排布区一侧显示 `↻ 顺时针`（direction=1）或 `↺ 逆时针`（direction=-1），带轻微旋转动效。
- **下家箭头**：在当前玩家卡片 → 下一位存活玩家卡片之间画一个高亮箭头 `▶` / `→`，并在下家卡片上加“下家”角标。
- **实时更新规则**（关键，必须随引擎状态刷新）：
  - **正常轮转**：箭头指向 `nextAliveIdx(currentIdx)`。
  - **超车**：被跳过者卡片盖“跳过”灰罩，箭头**越过**他指向下下家（与 `advanceTurn` 中 `skippedPlayerId` 双跳逻辑一致）。
  - **掉头**：`direction *= -1` 后，所有箭头整体反向；2 人局掉头=自己再来一轮，箭头指向自己并显示“再来一轮”。
  - **连续油门**：当前高亮锁定在 `doublePlayTarget` 直到 `doublePlayRemaining` 归零；期间下家箭头指向该目标。
  - 以上每次 `renderAll()` 都重算，确保“超车/掉头/连续油门后实时更新”。

#### 3.1.4 轮胎耐久可视化 【P0】
**区域**：`#racers` 内每个 `.racer`（`renderRacers` 改造）

- 用**分段耐久条**替代纯 `🛞` emoji：每个玩家一条横条，分 3 格（`START_TIRES=3`），满格=`#0a9c7c`、空格=灰 `#ccc`，格间留缝。
- 保留 emoji 作点缀（可选），但主信息靠耐久条；淘汰者整条置灰 + “淘汰”标。
- 规格：条宽 ≈ 卡片宽 80%，高 8px，圆角，3 段等宽。

#### 3.1.5 模式状态常驻栏 【P0】
**区域**：`#screen-game` 顶部新增 `#mode-bar`（**新 DOM**）
**目的**：彻底替代 `showFloatingTip` 的 2 秒消失提示，让模式状态常驻可见。

| 状态 | 常驻内容 |
|---|---|
| 方向 | `↻ 顺时针` / `↺ 逆时针`（实时随掉头翻转） |
| 连续油门 | `⚡ 连续油门 → {target名}（剩 {N} 张）`，N=`state.doublePlayRemaining` |
| 升档激活 | `⬆ 升档激活：下张速度牌额外出`（`state.extraSpeedMode` 为真时显示） |
| 你的回合 | 轮到玩家时显示 `▶ 你的回合` |

- 无相关状态时栏收起或显示“常规轮转”，不占视觉噪音。
- **删除** `showFloatingTip` 的玩法提示用途（保留其作为“事件横幅”的轻量实现可复用，见 3.1.6）。

#### 3.1.6 关键事件横幅 【P1】
**区域**：`#screen-game` 新增 `#event-banner`（**新 DOM**），复用 `showFloatingTip` 的定位样式但**常驻至下一玩家动作**（或 4 秒后淡出，取较晚者）。

必须弹横幅的事件（在 `playCard` / 爆胎 / 淘汰处触发）：
- `⚠️ {name} 打出【连续油门】，{target} 需连出 2 张！`
- `💥 {name} 爆胎！速度归零，轮胎 -1（剩 {n}）`
- `↺ 掉头！出牌方向反转`
- `🏎️💨 {name} 超车，跳过 {skipped}`
- `🔄 失控反转：{old} → {new}`
- `🏆 {name} 胜利 / 全部出局`

横幅与日志（`#log`）并存：日志保留完整流水，横幅只抓“会改变局势的关键事件”。

### 3.2 规则理解降门槛

#### 3.2.1 功能牌统一 icon + 一行效果标签系统 【P0/P1】
**区域**：手牌功能牌（`renderHand`）+ 规则书弹层（`#rule-modal`）
**目的**：9 种功能牌一眼可辨、且知道“打出会怎样”。

| 功能牌 | 统一 icon | 一行效果标签 | 何时用（建议） |
|---|---|---|---|
| 刹车 | 🛑 | 减速 −10 | 速度偏高、想留余量时 |
| 急刹 | 🛑💢 | 急减速 −20 | 接近爆胎线、急需大幅降速 |
| 超车 | 🏎️💨 | 跳过下家 | 下家快赢/想抢节奏时 |
| 掉头 | ↩️ | 反转方向 / 2人再来一轮 | 想改变出牌顺序、2人局白赚一轮 |
| 连续油门 | ⚡⚡ | 下家连出 2 张 | 想给下家（尤其玩家）制造压力 |
| 加油 | ⛽ | 加速 +18 | 速度低、想快速拉高时 |
| 调速50 | 🎯 | 速度设为 50 | 速度过高/过低，想直接重置到中段 |
| 升档加速 | ⬆️ | 额外出 1 张速度牌 | 手里有高数值速度牌、想连打爆发 |
| 失控反转 | 🔄 | 速度 = 101 − 当前 | 速度低时翻到高位、速度高时翻到低 |

- 手牌功能牌卡面统一显示：`[icon] 名称` + 一行小字效果标签（替换现有仅 `icon + name`）。
- 颜色沿用现有 `.func-*` 配色，不新增。

#### 3.2.2 规则卡弹层（点卡看说明）【P1】
**区域**：新增 `#rule-modal`（**新 DOM**，复用现有 `.overlay`/`.modal` 样式）

- 手牌中**任意功能牌可点击弹出**该牌的规则卡：大 icon + 名称 + 一行效果 + “何时用” + 一个迷你示例（如“当前 60，打出急刹 → 40”）。
- 速度牌点击也可弹“速度牌：直接累加到当前速度”。
- 主菜单/游戏内加 `? 玩法` 按钮，打开**完整规则书**（9 张功能牌总表 + 爆胎/淘汰/胜负说明）。
- 实现：`renderHand` 中给功能牌 `div` 绑定 `onclick` 时区分“出牌”与“看说明”——建议**长按/双指**看说明、**单击**出牌；或卡面右上角加 `?` 小按钮专用于看说明（移动端更稳，推荐此方案）。

#### 3.2.3 新手 3 步互动引导 【P1】
**区域**：新增 `#tutorial-overlay`（**新 DOM**），首局触发

- 触发：localStorage 无 `speed101_tutorial_done` 时，进入首局自动播放；主菜单加“玩法教学”可重看。
- 3 步（每步高亮对应区域 + 一句说明 + “下一步/跳过”）：
  1. **目标**：指向速度表 → “别让速度超过 **101**（红线）。超过就爆胎扣 1 轮胎，3 个轮胎用完淘汰。”
  2. **读卡**：指向手牌 → “出牌前看卡上『到 X』：绿=安全、橙红闪=危险、灰=会爆胎。优先出绿色。”
  3. **功能牌**：指向一张功能牌 → “功能牌各有绝招，点卡上 `?` 看说明。现在出一张试试！”（等待玩家完成首次出牌后引导结束，置 `speed101_tutorial_done`）。

### 3.3 AI 意图可视化

#### 3.3.1 AI 思考气泡 【P2】
**区域**：每个 AI `.racer` 上方新增 `.ai-bubble`（**新 DOM**，浮于卡片上方）

- **中/困难模式**显示：AI 回合前（即 `aiLoop` 调度时）展示一句意图气泡，如：
  - `AI2 在犹豫是否超车…`
  - `AI3 准备减速避险`
  - `AI1 想把速度顶到 95 逼你爆胎`
- **简单模式**：默认**不显示**；设置项“显示 AI 思考”可手动开启（满足“简单模式可关”）。
- **数据钩子（给程基岩）**：当前 `chooseAiCard(p)` 只返回 `idx`。需重构为返回 `{ idx, intent }`，`intent` 为上述短句，由各策略分支在选牌时一并产出（如走到“困难 AI 顶速度”分支 → intent=`想顶高逼你爆胎`）。气泡在 `playCard` 执行后淡出。
- 气泡停留 ~1.2s 再出牌（与现有 `setTimeout(aiLoop, 320+rand)` 节奏协调），既给玩家学习线索又不拖慢节奏。

## 4. 落地优先级清单（P0 / P1 / P2）

> 复杂度：低=纯 CSS/文案；中=需改渲染函数+少量状态；高=需改引擎返回结构或加新 DOM 系统。

| 优先级 | 项 | 区域 | 预期体验收益 | 实现复杂度 | 依赖 |
|---|---|---|---|---|---|
| **P0** | 手牌「预计到达速度」三态 | D | 出牌前即可预判安全/爆胎，误操作↓，规则门槛↓ | 中 | 复用 `state.speed`；改 `renderHand` |
| **P0** | 速度表爆胎红线+余量+区带 | C | “离爆胎还有多少”一眼可见，危险感知↑ | 低 | 改 `renderSpeedometer` + CSS |
| **P0** | 下家箭头+方向环 | B | 看清轮到谁、下家是谁、顺逆方向 | 中 | 改 `renderRacers`；用 `direction`/`nextAliveIdx` |
| **P0** | 轮胎分段耐久条 | B | 耐久感清晰，淘汰状态明确 | 低 | 改 `renderRacers` |
| **P0** | 模式状态常驻栏 | A | 连续油门/升档/方向不再“错过” | 低 | 新 `#mode-bar`；删 `showFloatingTip` 玩法提示 |
| **P0** | 功能牌统一 icon+一行标签 | D | 9 牌一眼可辨“打出会怎样” | 低 | 改 `renderHand` 文案 |
| **P1** | 关键事件横幅 | E | 关键局势变化不再漏看 | 低 | 新 `#event-banner`；在 `playCard`/爆胎处触发 |

## 5. 待用户确认项（决策点）

1. **必爆牌的交互方式**（§3.1.1）：方案原文要求“灰禁用”。但引擎允许主动爆胎，且存在“手里全是必爆牌”的极端情况（见 §6 边缘情况 #1），硬禁用会**软锁**。建议采用 **灰+红闪标注 + 点击弹确认“这张会爆胎，确定出？”**，既清晰又不断操作。请确认：
   - (a) 严格按原文硬禁用（接受极端软锁风险）；
   - (b) **推荐**：灰标 + 确认弹窗（默认）。
2. **危险阈值**：手牌三态与速度表危险区统一取 **85**（对齐 AI `danger>=85`）。若你希望更保守（如 90），请告知——需同步改 `renderSpeedometer` 的脉冲阈值与区带分割。
3. **简单模式 AI 气泡默认**：建议默认**关**，设置项可开。是否同意？
4. **可访问性文档**：当前 `design/` 无 `accessibility-requirements.md`。本方案已内嵌“不靠颜色 alone”原则，但是否需要我后续补一份独立可访问性需求文档（色盲/字体缩放/对比度）？

## 6. 边缘情况（设计需覆盖）

| # | 场景 | 处理 |
|---|---|---|
| 1 | 手里全是必爆牌（无安全可选） | 必爆牌仍可点（确认弹窗），避免全禁用导致软锁（见 §5-1） |
| 2 | `S' = 101` 恰好临界 | 引擎 `>101` 才爆 → 标“危险（临界）”而非“必爆”，与引擎一致 |
| 3 | 2 人局掉头 | 方向反转 + 下家箭头指向自己 + “再来一轮”标，不指对手 |
| 4 | 连续油门目标爆胎/淘汰 | `doublePlayTarget` 清空，`#mode-bar` 连续油门项立即消失，横幅提示“连续油门中断” |
| 5 | 升档激活但手牌无速度牌 | 引擎自动结束 `extraSpeedMode`，UI 同步隐藏升档徽章 |
| 6 | 调速50 | `S'=50` 恒安全（<85），不受当前速度影响 |
| 7 | 多张必爆并存 | 全部灰标，但至少允许一张确认出（不全局锁死） |
| 8 | 色盲/低视力 | 三态除颜色外必须有图标(✓/!/✕)+文字，对比度达 WCAG AA |

## 7. 实现备注（给程基岩的代码钩子）

**新增 DOM（均在 `#screen-game` 内）**
