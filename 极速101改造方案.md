## 0. 摘要（TL;DR）

- **核心循环**：轮流出牌 → 报出新的「速度值」→ 不能超过 101 → 超过则「爆胎」扣 1 个轮胎并速度归零 → 谁先丢光 3 个轮胎谁淘汰，最后存活者获胜。
- **本期形态**：玩家一人，对手全部为 AI，**开局可选 1–5 个 AI**（2–6 人局的数字化还原）。
- **技术**：纯前端、零构建、单 `index.html` + 原生 JS/CSS/SVG。itch.io 静态 HTML 托管，无需后端。
- **美术**：卡通赛车/公路风，仪表盘式速度表，小车生命条，高对比数字卡。
- **已完成功能**：洗牌发牌、出牌逻辑、爆胎判定、3 档 AI、功能牌（加速/刹车/急刹/升档/超车/极速50）、音效、胜负结算。

## 2. 已实现的游戏机制

### 2.1 规则引擎
- 3 个轮胎生命，3 张起始手牌，速度上限 101。
- 速度牌：直接加减；升档状态下翻倍。
- 功能牌：触发对应效果，处理额外回合与状态标记。
- 爆胎：速度 > 101 → 扣 1 轮胎、速度归零、清除升档。
- 淘汰：轮胎耗尽。
- 胜负：仅剩 1 人存活时结束，弹出结算。

### 2.2 AI 三档
- **简单**：从合法牌中随机出，不主动进攻。
- **中等**：危险区（≥85）优先减速/救急；会用升档、超车扩大优势。
- **困难**：在 ≥80 时主动把速度顶高逼玩家爆胎，更会配合升档打爆发。

### 2.3 界面
- 仪表盘式速度表，指针随速度移动，≥90 红区脉冲警告。
- 顶部一排小车展示玩家/AI 生命与当前回合。
- 底部玩家手牌，只有到自己回合才可点击。
- 滚动日志记录每步动作与爆胎事件。
- Web Audio 合成简单音效（出牌、爆胎）。

## 4. itch.io 发布步骤

1. 注册/登录 itch.io 开发者账号。
2. Create new project → **Kind of project: HTML**。
3. 将 `speed101/` 目录打包为 `speed101.zip`（根目录含 `index.html`）。
4. 在 Uploads 上传，勾选 **This file will be played in the browser**。
5. 设置：
   - 标题：`极速101 · 数学竞速`（英文：`Speed 101`）
   - 标签：`puzzle` `math` `educational` `card` `racing` `family`
   - 分类：Games → Puzzle
   - 价格：Free，可开「Pay what you want」
6. 封面：315×250 卡通小车仪表盘图；截图 3–5 张。
7. 发布后站内 iframe 即点即玩。
